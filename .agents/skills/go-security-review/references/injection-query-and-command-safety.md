# Injection, Query, And Command Safety Review

## When To Load
Load this when changed Go code builds SQL, search filters, datastore operators, shell commands, subprocess invocations, templates, HTML responses, URL/query strings for interpreters, or any command-like mini-language from user-influenced data.

## Attacker Preconditions
- The attacker can influence a value that reaches a query, command, template, expression parser, filter DSL, or external program argument.
- The code combines that value with interpreter syntax, an identifier, an operator, an executable name, or an option.
- The interpreter has access to sensitive data, system files, network calls, or state-changing behavior.

## Review Signals
- SQL uses `fmt.Sprintf`, string concatenation, template assembly, or raw filters with user-controlled values.
- Dynamic table, column, sort, or operator names are accepted without mapping to code-owned allowlists.
- `exec.Command` is called with user-controlled executable names, option names, shell strings, or `sh -c` / `cmd /c`.
- Code treats `cmd.String()` as shell-safe or reuses it as command input.
- `html/template` safety is bypassed with `template.HTML`, `template.JS`, `template.URL`, or by switching to `text/template` for HTML output.
- Error responses or telemetry include raw SQL text, command output, or interpreter diagnostics with sensitive values.

## Bad Finding Examples
- "This might be SQL injection."
- "Do not use exec with user input."
- "Templates are risky."

These are too broad unless the review shows the attacker-controlled value, the interpreter context, and the concrete correction.

## Good Finding Examples
- "[critical] [go-security-review] internal/infra/postgres/search.go:39 Axis: Injection And Query Safety; `fmt.Sprintf` inserts the request `filter` directly into the SQL `WHERE` clause. An authenticated caller can change the predicate and read rows outside the intended search. Replace the raw filter string with typed filter fields, use driver placeholders for values, and map any dynamic operators from a fixed allowlist."
- "[high] [go-security-review] internal/app/export.go:74 Axis: Injection And Command Safety; the export command runs `sh -c` with a filename derived from the request. A caller who controls the filename can influence shell parsing under the service account. Avoid the shell, hardcode the executable, pass validated operands as separate `exec.CommandContext` args, and bound the command with the request context."
- "[medium] [go-security-review] internal/web/render.go:31 Axis: Template Injection/XSS; the change wraps user-authored profile HTML in `template.HTML`, bypassing `html/template` contextual escaping. A profile editor can inject active markup into every viewer's page. Store the value as plain text or run it through an approved sanitizer before marking it trusted."

## Smallest Safe Correction
- Use parameter placeholders for SQL values and pass values as `QueryContext` or `ExecContext` arguments.
- For dynamic identifiers or sort directions, map request enums to internal constants; do not parameterize identifiers by string concatenation.
- Prefer existing Go APIs over subprocesses. If a subprocess is required, hardcode the executable, pass args separately, use `exec.CommandContext`, set a bounded context, and validate operands with an allowlist.
- Avoid `sh -c` and `cmd /c` for user-influenced operations. If unavoidable, escalate because escaping becomes policy-sensitive.
- Use `html/template` for HTML output and treat template authors as trusted but data as untrusted. Avoid typed safe wrappers unless the value is generated by trusted code or an approved sanitizer.

## Validation Ideas
- Add tests that prove special characters remain data, not syntax, for SQL values and command arguments.
- Add tests for unsupported dynamic identifiers, operators, and sort keys.
- Add context-timeout tests for subprocess paths.
- Add rendering tests that assert user text is escaped and typed safe wrappers are not used on untrusted data.
- Run targeted package tests; run `make go-security` when query, command, or templating surfaces changed.

## Repo-Local Anchors
- This repo uses sqlc-generated query files under `internal/infra/postgres/sqlcgen`; handwritten query changes deserve extra scrutiny.
- `Makefile` exposes `make go-security` for `govulncheck` and `gosec` and `make sqlc-check` for generated query drift.
