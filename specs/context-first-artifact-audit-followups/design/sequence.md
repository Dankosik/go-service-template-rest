# Sequence

1. Create this task-local follow-up bundle.
2. Patch `docs/spec-first-workflow.md` for central rules.
3. Patch session and deeper skill guidance to mirror those rules.
4. Run repository validation checks.
5. Update this bundle's task and closeout state from actual proof.

Failure points:

- If central docs and skill guidance diverge, repair the skill wording to point back to central authority instead of creating a second authority.
- If validation reports stale generated or mirrored skill surfaces, run the repository-owned sync command and re-check.
- If a proposed edit creates a universal template or new artifact type, stop and keep the change as guidance on existing artifacts instead.
