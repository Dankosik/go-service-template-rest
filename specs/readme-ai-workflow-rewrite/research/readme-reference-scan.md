## Takeaways
- Strong AI-native READMEs lead with operator outcomes, not dependency lists.
- Solo-friendly repos use “you”, “your agent”, or clearly personal framing instead of organization-speak.
- Capability bullets near the top help readers orient before the README dives into workflow or commands.
- Workflow-heavy repos explain control surfaces explicitly:
  - who orchestrates;
  - what specialists or stages do;
  - when delegation happens;
  - which artifacts and commands prove progress.
- Tables work well when grouped by category rather than presented as one giant catalog.

## Adaptation For This Repository
- Use Codex and Aider as the reference for a clean hero and quickstart.
- Use Agent Orchestrator and Agent Teams Lite as the reference for workflow explanation.
- Use OpenClaw and LLM Council as the reference for solo-builder tone.
- Use Serena as the reference for capability-led bullets near the top.
- Use Spec Kit and Awesome Claude Skills as the reference for grouped tables and taxonomy.
- Keep the Go/OpenAPI/Postgres stack visible, but secondary to workflow.

## Risks To Avoid
- Over-selling the repo as a general-purpose multi-agent platform.
- Copying a product-marketing tone from AI repos that are trying to sell a hosted product.
- Making the skills section feel like a marketplace directory instead of a repository-native toolkit.
- Listing roles and skills without explaining the difference between them.
